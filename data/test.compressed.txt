## AI Overview
AI simulates human intelligence in machines: learning, reasoning, perception, language. Trained on data, not explicit rules.

Types:
- Narrow AI (ANI): task-specific (facial recognition, search, driving) — all current AI
- General AI (AGI): theoretical human-level cross-domain reasoning
- Generative AI: creates text/images/video/code from patterns (e.g., ChatGPT)

How it works: Data collection → ML training → neural networks → optimization (backpropagation/gradient descent) → inference.
- Supervised: labeled data | Unsupervised: hidden patterns | Reinforcement: reward/penalty trials
- Deep learning: multi-layer neural nets for complex pattern recognition

Enablers: big data, GPUs, Transformer architecture (2017+)

Applications: virtual assistants (Siri, Alexa), recommendations (Netflix, YouTube), computer vision, generative tools (ChatGPT), healthcare diagnostics.

## LangSmith
Framework-agnostic platform for debugging, evaluating, and monitoring LLM apps.
- Tracing: visual run tree, step-by-step execution capture
- Evaluation: datasets + AI-based/custom evaluators
- Monitoring: real-time latency, cost, success-rate dashboards
- Prompt management: playground + version control

Integrates with LangChain, OpenAI, Anthropic, CrewAI.
Plans: Developer (5k traces/month free) → Plus/Startup → Enterprise. SOC 2 Type 2, GDPR, HIPAA compliant.

## Confluence AI Agent

### Native (fastest)
Enable Atlassian Intelligence / Rovo in admin settings, or build a Forge Rovo Agent using the Confluence REST API.

### Custom RAG Pipeline
1. **Auth:** Generate Atlassian API Token
2. **Ingest:** Confluence REST API (`/wiki/rest/api/content`) — use LangChain/LlamaIndex ConfluenceLoader
3. **Parse:** BeautifulSoup on `body.storage.value` HTML
4. **Embed:** OpenAI `text-embedding-3-small` → vector DB (Pinecone / ChromaDB / Milvus)
5. **Chat:** Streamlit or Slack bot → retrieve chunks → LLM (GPT-4 / Claude) generates answer

Low-code alternatives: MCP Server for Atlassian, or n8n (Confluence API node + AI Agent node).

### Microsoft Teams Integration
- **Low-code:** Copilot Studio → add Confluence data source → publish to Teams channel
- **Pro-code:** Teams Toolkit (VS Code) + Teams AI Framework + LangChain backend → deploy to Azure

### AWS Architecture
1. Store Confluence credentials in **AWS Secrets Manager**
2. **Bedrock Knowledge Base** — Confluence connector → Titan embeddings → OpenSearch Serverless
3. **Bedrock Agent** (Claude) linked to Knowledge Base with behavior instructions
4. **AWS Chatbot** → authorize Teams workspace → map channel to Bedrock Agent alias
5. Trigger in Teams: `@aws ask [agent-name] [question]`

For complex pre-processing: Lambda + EventBridge to pull Confluence REST API → S3 → trigger KB ingestion.
